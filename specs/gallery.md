# Gallery

## Purpose

Browse all images ever generated by the tool, with full metadata visibility and filtering capabilities.

## Layout

- Grid view of generated images (thumbnails)
- Images grouped by generation run by default (all images from one "Generate" click appear together)
- Most recent runs shown first
- **Virtual scroll** — all runs are present in the list logically, but only visible items are rendered in the DOM; handles large collections (thousands of images) without performance degradation

## Image Card

Each image in the gallery shows:
- Thumbnail
- Model name
- Prompt (truncated, expandable)
- Timestamp
- Key settings (e.g., steps, CFG, seed)

## Filtering & Search

- Filter by model
- Filter by date range
- Search by prompt text
- Filter by run (see all images from a specific run)

## Multi-Select & Deletion

- Each image card has a checkbox (visible on hover, or always visible in selection mode)
- Selecting any image enters **selection mode** — a toolbar appears showing the count of selected items and a **Delete** button
- "Select all" option to select all currently visible/filtered images
- Deleting removes the image files from disk and their records from the database
- A confirmation dialog is shown before deletion listing how many images will be deleted
- Deletion is permanent and cannot be undone

## Detail View

Clicking an image opens a detail panel showing:
- Full resolution image
- Full prompt and negative prompt
- All settings used (including seed)
- Model name and version
- Generation time
- Timestamp
- Link to view the full run it belongs to
- Download button
- **"Reproduce" button** — navigates to the Generator page with all fields pre-filled exactly as they were at the time of generation: prompt, negative prompt, aspect ratio, model selection (only that model checked), and all model settings including the exact seed. The user can then click Generate to reproduce the image or tweak settings from that starting point.

## Run View

Clicking a run shows all images generated in that run side by side — identical to the comparison view in the Generator page, but read-only and pulled from history.

- Each image in the run view also has a **"Reproduce"** button for that individual image (same behaviour as in the detail view)
